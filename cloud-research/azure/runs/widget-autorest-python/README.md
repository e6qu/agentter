# Build: Azure AutoRest (legacy) — Swagger 2.0 → Python

**Result: SUCCESS.** AutoRest generated a Python SDK from a minimal Swagger 2.0
spec. This is Azure's **legacy** codegen stack (deprecated; retires 2026-07-01).
See `../widget-typespec-ts/` for the **new** TypeSpec stack and `../README.md`
for the dual-stack overview.

## Toolchain
- Node 26 + `autorest` CLI 3.8.0. AutoRest self-downloads its core
  (`@autorest/core` 3.10.9), the `@autorest/python` 6.50.3 plugin, and
  `@autorest/modelerfour` 4.23.7 at runtime.
- **Live deprecation evidence:** at runtime AutoRest printed
  `⚠️ AutoRest is deprecated and will be retired on July 1, 2026. Please migrate
  to TypeSpec` — the same retirement the deliverable cites from the repo readme,
  observed first-hand.

## Input (`input/`)
- `widgets.json` — a hand-written Swagger 2.0 with one `GET /widgets/{widgetName}`
  operation and a `Widget` definition (incl. an `x-ms-enum` `color`).

## Commands (`cmd.sh`)
```
npm install autorest
npx autorest --python --input-file=widgets.json --output-folder=out-python --namespace=widgetservice
```

## Output (`output/out-python/`)
18 files, incl. sync + async (`aio/`) clients, `_configuration.py`,
`_utils/serialization.py`, `operations/_operations.py`.

## The key evidence (architecture confirmations)
- **Generation header verbatim:** `# Code generated by Microsoft (R) AutoRest Code
  Generator (autorest: 3.10.9, generator: @autorest/python@6.50.3) Changes may
  cause incorrect behavior and will be lost if the code is regenerated.`
- **The `_patch.py` escape hatch** is emitted at every level (`_patch.py`,
  `aio/_patch.py`, `operations/_patch.py`, `aio/operations/_patch.py`) — the
  documented "last-resort customization that survives regeneration" seam.
- Generated code imports `azure.core` (the hand-written runtime) — the
  thin-generated-surface-over-hand-written-runtime pattern.

## Takeaway
A faithful reproduction of the legacy AutoRest path, including the live
deprecation notice — half of Azure's "dual stack" made concrete.
